# Ideatask 主看板 UI 修改需求说明

## 1. 本次修改目标

当前主看板页面需要进一步简化。

主看板的核心目标不是展示所有细节，也不是展示项目内部的导图结构，而是让用户一眼看到：

1. 我现在正在执行哪些项目
2. 我脑子里还有哪些尚未开始执行的想法

因此，主看板页面只保留两个核心区域：

- 正在执行的项目
- 想法库

页面不需要显示每个项目的具体子节点，也不需要显示今日重点、任务列表、统计图、复杂底部模块等内容。

主看板应该是一个非常简洁的“当前脑内状态总览页”。

---

## 2. 核心产品规则

### 2.1 正在执行的项目和想法库必须分开

“正在执行的项目”和“想法库”是两个不同阶段的内容。

- 正在执行的项目：已经开始推进的项目或主题
- 想法库：还没有开始执行的纯点子

这两个区域不能混在一起。

### 2.2 同一个主题不能同时出现在两个区域

这是本次修改最重要的产品规则。

如果一个想法已经开始执行，它就不应该继续出现在想法库里。

错误示例：

- “模型效果提升”已经是 Glantan AI 项目的 Next
- 但它又出现在想法库中

正确逻辑：

- 如果“模型效果提升”已经开始执行，它只属于正在执行的项目
- 如果它还只是一个点子，它只属于想法库
- 同一个主题不能重复出现在两个区域

### 2.3 想法库只保存未开始执行的点子

想法库中的内容必须是纯点子，状态只能是：

- 新想法
- 待整理
- 待评估

想法库中不应该出现以下状态：

- 进行中
- Active
- Doing
- Planning
- 已开始
- 执行中

一旦一个点子开始执行，它就应该从想法库移除，并转入正在执行的项目。

---

## 3. 页面整体结构

主看板页面结构建议如下：

1. 顶部导航栏
2. 简化筛选栏 / 视图切换
3. 正在执行的项目
4. 想法库
5. 底部中央快速记录按钮

页面不再需要：

- 今日重点模块
- 任务列表模块
- 项目子节点展开
- 项目导图连接线
- 数据统计面板
- 复杂底部信息区
- minimap

---

## 5. 筛选栏和视图切换

筛选栏应简化，不要显示过多内容。

### 5.1 左侧筛选

建议只保留：

- All
- Projects 6
- Ideas 7

可以暂时去掉：

- Tasks
- Issues

原因：

主看板的核心不是任务管理页，也不是问题管理页，而是“项目 + 想法”的总览。

Tasks 和 Issues 可以在项目详情页或 List 视图里查看。

### 5.2 右侧视图切换

保留：

- Dashboard
- List
- Mindmap

默认选中：

Dashboard

不同视图的作用：

- Dashboard：主看板，一眼看当前项目和想法库
- List：列表模式，适合筛选和批量管理
- Mindmap：导图模式，适合进入项目后进行结构化思考

## 6. 第一核心区域：正在执行的项目

### 6.1 区域定位

“正在执行的项目”是页面上方的主要区域。

它显示当前已经开始推进的项目或主题。

这里不显示项目内部的子节点，只显示项目级摘要信息。

### 6.2 区域标题

标题：

正在执行的项目

副标题：

6 个项目

左侧可以使用 folder / project 图标。

### 6.3 项目卡片展示规则

每个项目卡片只显示摘要信息，不展开详情。

项目卡片包含：

- 项目图标
- 项目名称
- 状态
- ideas 数量
- tasks 数量
- Next 当前下一步
- 简单进度条

### 6.4 项目卡片不应该显示的内容

不要在主看板项目卡片里显示：

- 完整子任务列表
- 具体子节点
- 连接线
- issue 详情
- note 详情
- 项目内部导图
- 今日任务列表

这些内容应该进入项目详情页后再查看。

## 7. 第二核心区域：想法库

### 7.1 区域定位

“想法库”是页面下方的第二核心区域。

它只保存尚未开始执行的点子。

这些点子可能未来会变成项目，也可能会被删除、归档或继续等待。

### 7.2 区域标题

标题：

想法库

副标题：

7 个想法

左侧可以使用 lightbulb / bulb 图标。

### 7.3 想法库展示规则

每条想法只显示非常简洁的信息：

- 灯泡图标
- 想法标题
- 分类标签
- 状态标签

不要显示过多信息。

### 7.4 想法库状态规则

想法库中的状态只能是：

- 新想法
- 待整理
- 待评估

不能出现：

- 进行中
- Active
- Doing
- Planning
- 已开始
- 执行中

## 8. 排他规则

这是前端实现时必须遵守的规则。

### 8.1 内容阶段

每个主题只能处于一个阶段：

- idea
- project
- archived

如果处于 idea 阶段，显示在想法库。

如果处于 project 阶段，显示在正在执行的项目。

如果处于 archived 阶段，默认不显示在主看板。

### 8.2 想法转项目规则

当用户把某个想法转为项目时：

1. 从想法库移除
2. 新增到正在执行的项目
3. 状态变为 Active 或 Planning
4. 可以生成项目卡片
5. 原想法不再作为独立想法显示在主看板

### 8.3 项目暂停规则

如果项目暂停：

- 仍然可以保留在正在执行项目区域
- 状态显示为 Paused
- 不应该退回想法库

因为它已经不是纯点子，而是已经启动过的项目。

### 8.4 项目归档规则

如果项目完成或放弃：

- 从正在执行项目区域移除
- 进入归档区域
- 默认不在主看板显示

## 9. 页面交互规则

### 9.1 点击项目卡片

点击项目卡片后，进入项目详情页。

项目详情页可以显示：

- 项目导图
- 项目任务
- 项目问题
- 项目备注
- 项目时间线
- AI 拆解
- 项目历史

但这些不出现在主看板。

### 9.2 点击想法条目

点击想法条目后，打开想法详情。

想法详情可以显示：

- 想法描述
- 标签
- 创建时间
- AI 评估
- 转为项目
- 归档
- 删除

### 9.3 Capture a thought 按钮

底部中央保留一个主要按钮：

Capture a thought

点击后打开快速输入。

快速输入可以支持：

- 记录新想法
- 选择分类
- 设置状态：新想法 / 待整理 / 待评估
- 以后可以支持 AI 自动分类

默认情况下，通过 Capture a thought 新建的内容应该进入想法库，而不是直接进入正在执行的项目。

除非用户明确选择“转为项目”或“开始执行”。

---

## 10. 页面底部规则

页面底部应该保持干净。

保留：

- 一个居中的紫色 Capture a thought 按钮

不要显示：

- 今日重点
- 任务列表
- 复杂统计
- 时间线
- 多余操作说明
- minimap
- 画布控制提示

主看板不是导图画布，因此不需要 minimap 和拖动画布操作提示。

---

## 11. 视觉层级

页面视觉重点顺序应该是：

1. 正在执行的项目
2. 想法库
3. Capture a thought

顶部筛选和视图切换只是辅助，不应该抢视觉重点。

---

## 12. 信息密度控制

主看板要低信息密度。

每个项目卡片只保留 5 类信息：

- 名称
- 状态
- 数量
- 下一步
- 进度

每个想法条目只保留 4 类信息：

- 标题
- 分类
- 状态
- 图标

不要在主看板放太多描述文本。

## 16. 本次修改前后对比

### 修改前

页面尝试展示：

- 项目
- 想法
- 任务
- 今日重点
- 项目内部事项
- 多个辅助模块

问题：

- 信息太多
- 正在做的项目和纯点子没有彻底分开
- 同一个主题可能重复出现
- 主看板不够简洁
- 用户不能一眼看清当前脑内状态

### 修改后

页面只展示：

- 正在执行的项目
- 想法库

改进：

- 项目和点子彻底分开
- 同一个主题不重复出现
- 想法库只保存未开始的点子
- 页面底部保持干净
- 用户一眼就能看清正在做什么和还想做什么

## 17. 最终设计原则

主看板不是所有信息的集合页。

主看板只回答两个问题：

1. 我现在正在执行哪些项目？
2. 我还有哪些尚未开始的想法？

其他所有细节都应该点开后再看。

---

## 18. 一句话总结

Ideatask 主看板应该是：

一个极简的当前状态总览页，上方是正在执行的项目，下方是未开始的想法库；想法一旦开始执行，就从想法库移除，进入正在执行的项目。
